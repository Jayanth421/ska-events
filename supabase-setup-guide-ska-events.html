🗄️ Supabase Setup Guide

Follow these steps in order. Takes about 5 minutes total.

1
Open the SQL Editor in Supabase

Log in to your Supabase dashboard, then navigate to:

supabase.com/dashboard → Your Project → SQL Editor → + New Query

You will see a blank query editor. This is where you will paste and run the SQL below.

2
Run the Full Schema SQL

Open the file src/lib/supabase/schema.sql from your project, copy the entire file, paste it into the SQL Editor, and click Run.

⚠️ If you already ran an older version of the schema before, that is fine — all statements use CREATE TABLE IF NOT EXISTS so they will not break existing data. Just run it again.

This will create all 8 tables:

languages EN / Telugu language settings
translations Key-value language translations
profiles User profile data
bookings Customer event booking submissions
events Admin-managed internal events
vendors Vendor/supplier directory
gallery_items Gallery photos managed from admin panel NEW
planning_packages Packages shown on the website, managed from admin panel NEW
✅ After running, you should see "Success. No rows returned" at the bottom of the editor.
3
Create Your Admin User

3a. Go to:

Your Project → Authentication → Users → Add User

Fill in:

  • Email: admin@skaevents.com (or any email you want)
  • Password: choose a strong password
  • ✅ Check "Auto Confirm" so you don't need to verify email

Click Create User.

3b. Now go back to SQL Editor and run this command to give that user admin rights. Replace the email if you used a different one:

UPDATE auth.users
SET raw_user_meta_data = raw_user_meta_data || '{"role":"admin","is_admin":true}'
WHERE email = 'admin@skaevents.com';
✅ You should see "1 row affected" after running this.
4
Verify Tables Were Created

Go to:

Your Project → Table Editor

You should see all 8 tables listed on the left side. If you see gallery_items and planning_packages in the list, everything worked correctly.

💡 You can also quickly verify by running this query in the SQL Editor:

SELECT table_name FROM information_schema.tables WHERE table_schema = 'public' ORDER BY table_name;
5
Log In to the Admin Panel

Start your dev server if it's not already running, then open:

http://localhost:3000/en/admin/login

Use the email and password you created in Step 3a. You will land on the Admin Dashboard with all modules including the new Gallery and Planning & Packages cards.

✅ Once logged in, go to Planning & Packages and add your packages — they will instantly appear on the public website. Go to Gallery and add your photos — they will instantly appear on the Gallery page.

Quick Reference — New Tables Only

🖼
gallery_items — what each column means
srcFull URL of the image (Unsplash, your Supabase Storage, Google Drive, etc.)
titleEnglish caption shown on hover
title_teTelugu caption (optional)
categoryweddings / engagements / birthdays / corporate / other
sort_orderLower number = shown first (0, 1, 2 …)
activetrue = visible on website, false = hidden
📦
planning_packages — what each column means
keyA unique short name like "premiumWedding" (no spaces)
title_en / title_tePackage name in English and Telugu
description_en / _teShort description text
price_en / price_tePrice string e.g. "₹2,50,000"
badge_en / badge_teCorner badge e.g. "Most Popular"
image_urlPackage card image URL
features_en / _teArray of bullet-point features (enter one per line in the admin form)
activetrue = shown on website
sort_orderDisplay order (0 = first)
⚠️ Already have the old tables? No problem. The old tables (bookings, events, vendors, etc.) already exist in your Supabase — running the schema again will skip them safely. Only gallery_items and planning_packages are brand new and will be created fresh.
